Bolero

Bolero is a domain-specific language (DSL) intended for easily building CRUD applications. Bolero is a superset of the Go programming language and is compatible with all Go codebases.

Bolero is currently in an early alpha phase. Below is a list of features that Bolero will bring, along with the current status of implementation.

Features

Enums

Bolero has first-class enum types. Proper enums allow safe switch statements.

Like structs, enums can define methods.

Enum values can be compared with == and != operators, but also with <, >, <= and >=. Range over enum is also supported. 

type Color enum {
    Red
    Green
    Blue
}

func (c Color) Hex() string {
    switch c {
        Red:
            return "F00"
        Green:
            return "0F0"
        Blue:
            return "00F"
    }
}

func Rose() Color {
    return Color::Red
}

type ItalianPerson struct {
    Name string {
        Title = "Nome",
        json = "name",
        MaxLength = 50
    }
    Age uint {
        Title = "Età",
        json = "age"
    }
}

func validate(p ItalianPerson) error {
    if len(p.Name) > p::Name.MaxLength {
        return errors.New("name is too long")
    }
}

Static properties

Each field in structure can have a list of constant values attached to it. These values are shared between all instances of that structure type.

These static properties can replace field tags, but can also provide much more.

Components

Components are not just used for building HTML. They can be used for defining PDF layouts, web services, or even parsers. 

comp VendorPage(vendor Vendor){
    Heading(){
        vendor.Name
    },
    FieldBox(){
        Field(Editable = false){
            vendor.Address
        },
        Field(){
            vendor.City
        },
        Field(MaxLength = 10){
            vendor.Vat
        }
    }
}

Query language

func FindTopCustomer() string {
    top, err := from Customer
        select name
        orderBy spending
        limit 1
}

Full stack framework

Bolero will have a full-stack framework that will maximally utilize the listed features. This framework will enable simple development of all CRUD-based business applications such as ERP, WMS, back offices, etc.

The framework will support localization, authorization, background tasks, logging, retention policies, debugging, custom web services, and auditing.

FAQ

Why Go?

Go is a language with amazing performance, exceptional deployability, and an ever-growing ecosystem. Go's non-breaking guarantee makes it an ideal foundation for making a DSL on top of it.

Can we include feature X?

Maybe. I am always open to ideas that will work toward this language's goals. Open an issue on the repo, and let's discuss your ideas.

There are still some things that I would like to introduce to the Bolero language, but I only have a vague idea of how to proceed. Features like default values for struct fields, or simple error handling.

Why use Bolero in age of AI?

While AI can quickly produce large amounts of code, AI is still far from producing consistent, correct code.

While Bolero is a perfect language to use with AI, it also tries to remove some of the need for AI. Namely, it tries to let the programmer express high-level ideas more concisely, on a more abstract, but still formal, level.
